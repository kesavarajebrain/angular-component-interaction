ngOnInit()

👉 ngOnInit() is one of Angular’s lifecycle hooks that runs once after the component is initialized — specifically after Angular has set all the input properties ( @Input) of the component.
It is a good place to put initialization logic, such as fetching data from a server or setting up default values for properties.
It is called only once during the component's lifecycle, making it ideal for tasks that need to be performed only once when the component is created.

Suppose you have a parent component passing a value via @Input():

In Parent component
⮜app-child [data]="'Hello Angular'" ⮞⮜/app-child⮞

Output :
Constructor: data = undefined ❌
ngOnInit: data = Hello Angular ✅

💡 That’s because @Input() bindings are not ready inside the constructor, but they’re available inside ngOnInit().

@Input() data!: string; 👉 👉 🔴 Definite Assignment Assertion operator.
@Input() data: string; // ❌ Error: Property 'data' has no initializer (without !)

By adding the exclamation mark (!), you’re saying to TypeScript:
“Trust me — this property will definitely be initialized before it’s used.”

What happens if you forget !
If you use strict mode ("strictPropertyInitialization": true in tsconfig.json):
@Input() data: string; // ❌ TS2564: Property 'data' has no initializer

Other ways to avoid the compilation Error
Either initialize it directly: @Input() data: string = '';
Or make it optional: @Input() data?: string;

Difference Between using (!) NOT in variables(before single/ before double/ after single) like this - !variable / !!variable / variable!

1. ! before a variable (JavaScript logical NOT)
let isLoggedIn = true;
console.log(!isLoggedIn);

2. !! (double NOT) → convert any value to boolean
let isLoggedIn = true;
console.log(!!isLoggedIn); // false
console.log(!!"Hello"); // true
console.log(!!0); // false
console.log(!!undefined); // false

3. ! after a variable → TypeScript “Definite Assignment Assertion”
@Input() data!: string;
