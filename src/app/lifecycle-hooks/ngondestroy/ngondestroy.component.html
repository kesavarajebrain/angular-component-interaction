ngOnDestroy()

ngOnDestroy() is a lifecycle hook that Angular calls just before the component is removed from the DOM (destroyed).

Parent Component
Why ngOnDestroy? 
✅ Counter stops immediately (because we cleared the interval).
✅ No memory leaks.
✅ Perfect cleanup.

⚠️ Why It’s Important?
If you don’t use ngOnDestroy properly:
🔹Subscriptions keep running
🔹Timers never stop
🔹Components hold memory references
🔹App becomes slow or crashes over time
🔹So this hook is essential for performance and stability.

What needs to clean?
✅ Clean up Subscriptions
✅ Clear Intervals or Timeouts
✅ Remove Event Listeners
✅ Destroy External Libraries
