Compilation

Angular converts your HTML + TypeScript into efficient JavaScript code that the browser can run.

Before going to angular compilation, first we need to understand normal webpage renders on browser.

How Browser renders a webpage?
Browser = HTML parser + CSS engine + JS engine (like V8) working together

🚀 Step 1 — You Enter a URL 

Example: https://example.com

What happens:
DNS lookup → finds server IP
Browser sends HTTP request
Server returns:
index.html

📄 Step 2 — HTML Starts Loading (Parsing Begins)

Imagine index.html its starts to render

1. HTML → DOM Tree

Browser reads HTML top to bottom
👉 Converts into DOM (Document Object Model)

html
 ├── head
 └── body
      ├── h1
      └── button

🎨 Step 3 — CSS Loading 

Browser sees:
⮜link rel="stylesheet" href="style.css"⮞

👉 It:
Downloads CSS
Builds CSSOM (CSS Object Model)
🧠 Step 4
Combine DOM + CSSOM → Render Tree 
👉 Browser creates:
  Render Tree = What actually needs to be shown
⚠️ Important — JavaScript is a BLOCKER 

When browser sees:

⮜script src="app.js"⮞⮜/script⮞
👉 It stops HTML parsing ❗

Why?
Because JS can modify HTML.
⚙️ Step 5 — JavaScript Execution (Core Part) 
Now enters:

👉 V8

🔥 What V8 actually does:

1. Parse JS → AST (Abstract Syntax Tree)
let a = 10;
Converted into internal structure

2. Compile → Machine Code
V8 uses:
Interpreter + JIT compiler (TurboFan)

3. Execute
Now JS runs.

Machine code = binary instructions (0s and 1s) that CPU understands directly

JIT (Just-In-Time) compiler = converts code into machine code during execution (runtime)

🚫 JIT is NOT for local build (like Angular build)

JIT is:
Inside browser
Happens automatically
Happens during runtime
🧩 Example — Real Flow
document.getElementById('btn').addEventListener('click', () => {
  alert('Clicked');
});
        
What happens:

JS runs
Finds button in DOM
Registers event listener
Does NOT execute immediately

👉 It waits for user action
🖱️ Step 6 — User Interaction (Event System) 

When user clicks:
👉 Browser sends event to: 🔄 EVENT LOOP 

Event Flow:
Click happens
Event goes to Event Queue
JS Engine checks Call Stack
Executes callback
🧠 Step 7 — DOM Updates 
Inside your function:

document.body.innerHTML = "Hi";

👉 Now:
DOM changes
Browser re-renders UI
🔥 Critical Understanding
Browser has 3 main engines: 

| Engine         | Role           |
| -------------- | -------------- |
| HTML Parser    | Builds DOM     |
| CSS Engine     | Styles         |
| JS Engine (V8) | Executes logic |
    
⚡ Where TypeScript Comes In

👉 Important:
Browser DOES NOT understand TypeScript
            
So what happens?

You write:
let a: number = 10;

👉 Build tools (like Angular/React/Webpack) convert it into:
let a = 10;

👉 THEN browser executes it via V8
1️⃣ JIT — Just-In-Time Compilation

💡 Idea:
Compile inside the browser at runtime

🔄 Flow (Step-by-step)

You write:

TypeScript
HTML template
Browser loads your app
Angular compiler runs in browser
Templates → converted to JS
App starts

📦 What gets shipped?
Your code
Angular compiler ❗

⚠️ Problems with JIT
Slower startup (compilation happens in browser)
Bigger bundle (compiler included)
Errors show at runtime 😬

🧪 Example Scenario

You write:
⮜div⮞user.name⮜/div⮞

But user is undefined

👉 In JIT:
App loads
Then crashes in browser

JIT is mostly used for:
development
debugging
2️⃣ AOT — Ahead-Of-Time Compilation

💡 Idea:
Compile before the app reaches the browser

🔄 Flow (Step-by-step)

You write:
TypeScript
HTML

During build (ng build):
Angular compiler runs on your machine

Templates → converted into JS
Final bundle is already compiled
Browser just executes it

📦 What gets shipped?
Only optimized JS
❌ No Angular compiler

🚀 Advantages
Faster startup ⚡
Smaller bundle 📦
Errors caught during build 🧠

🧪 Same Example
⮜div⮞user.name⮜/div⮞

👉 In AOT:
Build will FAIL ❌
You fix before deployment

👉 In Angular (latest versions):
AOT is default

Why AOT improves performance?

Because:
No runtime parsing of templates
No runtime compiler
Pre-generated instructions

👉 Browser directly executes optimized JS


| Concept                  | JIT        | AOT           |
| ------------------------ | ---------- | ------------- |
| When compilation happens | Browser    | Build time    |
| Who does it              | Client     | Build system  |
| Performance              | Slower     | Faster        |
| Bundle size              | Bigger     | Smaller       |
| Error detection          | Runtime 😬 | Build time 😎 |
    
🧑‍🍳 JIT = Cooking after customer orders
Customer comes → you start cooking
Takes time
Mistakes happen live

****************
🍱 AOT = Meal prep before opening restaurant
Everything ready before customer arrives
Fast serving
Errors already fixed
