Lazy Load

When you use lazy loading in Angular, you're basically telling the browser:
“Don’t download everything now. Download only when needed.”

  • Lazy loading = load component/module only when route is accessed
  • loadComponent replaces static import
  • loadChildren loads separate route files
  • Helps reduce initial bundle size
  • Improves initial load time
  • Useful for large apps (100+ components) ✅

Lazy Loading என்ன?
👉 Application open ஆகும் போது எல்லா components-யும் load பண்ணாமல்
👉 User எந்த page open பண்ணுறானோ அதுக்குத்தான் load பண்ணுவது

Single app.routes.ts is enough for everything?
Technically yes, but ❌ not scalable

🧠 Real-world:
We split routes like:

app.routes.ts
admin.routes.ts
client.routes.ts
forms.routes.ts

👉 This is still lazy loading — just better architecture

Old method : app.routing.module.ts 

{
    path:'home'
    component: HomeComponent
}
   
Angular bundles all imported component into main.js (File size is big)
Browser downloads that JS (on application starts)

எல்லா component code-மும் main.js-ல வரும்
App open ஆகும் போது எல்லாம் download ஆகும் 😓

New Method : app.routes.ts 

{
    path: 'home',
    loadComponent: () =>
    import('./home/home.component')
    .then(c => c.HomeComponent)
}
                
Angular bundles imported few components into main.js 
(For the supporting initial components, file size is less)
Browser downloads the chunk.js file when user route to the component

அந்த component மட்டும் load ஆகும்
Initial load fast ஆகும் ⚡
1. loadComponent (Small Unit Lazy Loading)
app.routes.ts

{
  path: 'profile',
  loadComponent: () =>
    import('./profile/profile.component')
      .then(c => c.ProfileComponent)
}
                    
Only ProfileComponent is loaded
Creates small chunk
Loads when user visits /profile

📦 Output:
main.js
profile.chunk.js
                
🟢 loadComponent (Simple)
👉 ஒரு single component மட்டும் lazy load

🔵 loadChildren (Important)
👉 ஒரு முழு feature (group of components) lazy load

loadComponent → One screen
loadChildren  → One feature (contains multiple screens)
2. loadChildren (Feature Lazy Loading)
app.routes.ts

{
  path: 'admin',
  loadChildren: () =>
    import('./admin/admin.routes')
      .then(r => r.ADMIN_ROUTES)
}

admin.routes.ts

export const ADMIN_ROUTES = [
  {
    path: '',
    loadComponent: () =>
      import('./admin.component').then(c => c.AdminComponent)
  },
  {
    path: 'adminusers',
    loadComponent: () =>
      import('./adminusers.component').then(c => c.AdminUsersComponent)
  }
];
    
User visits /admin
Angular loads:
👉 admin.routes.ts
Registers child routes
Then loads components as needed

📦 Output:
main.js
admin.chunk.js (contains all admin routes)
Bonus: Lazy Loading + NgRx Together
Problem without lazy loading:
All reducers/effects loaded at startup ❌
Store becomes heavy ❌

Route loads → Feature loads → Store registers → UI renders

👉 Store is NOT global anymore
👉 It becomes feature-based
Example :

⚙️ Step 1: Lazy Route (loadChildren)

{
  path: 'admin',
  loadChildren: () =>
    import('./admin/admin.routes').then(r => r.ADMIN_ROUTES)
}

❌ பழைய முறை:
App start → எல்லா reducer + effects load 😓

✅ புதிய முறை:
/admin open → admin store மட்டும் load 🔥

⚡ Why This is Powerful
✅ Performance: Smaller initial bundle , Faster startup
✅ Scalability: Each feature isolated
✅ Maintainability: Admin logic stays inside admin

With lazy loading:
Load feature only when route is opened ✅
Register reducers/effects on demand ✅

⚙️ Step 2: Feature Routes + Store

admin.routes.ts 
import { Routes } from '@angular/router';
import { provideState, provideEffects } from '@ngrx/store';
import { adminReducer } from './store/admin.reducer';
import { AdminEffects } from './store/admin.effects';

export const ADMIN_ROUTES: Routes = [
  {
    path: '',
    providers: [
      provideState('admin', adminReducer),
      provideEffects([AdminEffects])
    ],
    children: [
      {
        path: '',
        loadComponent: () =>
          import('./admin.component').then(c => c.AdminComponent)
      },
      {
        path: 'users',
        loadComponent: () =>
          import('./users.component').then(c => c.UsersComponent)
      }
    ]
  }
];
