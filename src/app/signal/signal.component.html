Signal

Signals are a new, built-in way for Angular to track state changes and automatically update your UI ‚Äî without relying on Zone.js or the old change detection system.

üß† Simplified Explanation

Before Signals:
  • Angular had to walk the whole component tree (change detection) to see what changed.
  • It relied on Zone.js to know when to start checking.
  • This caused unnecessary checks ‚Äî even when nothing really changed.
    • With Signals:
  • Each piece of data tells Angular directly when it changes.
  • Angular doesn‚Äôt need to check everything ‚Äî only what‚Äôs affected.
  • No Zone.js required. No over-checking.

So now, Angular becomes fine-grained reactive ‚Äî like Reactivity in Solid.js or Vue 3.

  • When you update that signal, it sends an electrical pulse üí• to Angular.

    ‚ÄúHey, count changed ‚Äî update only that part of the UI.‚Äù
  • ‚úÖ No need to run a full change detection cycle.

Signal is a replacement of the older change detection strategy?

‚öôÔ∏è 1Ô∏è‚É£ Before Signals: The ‚ÄúOld World‚Äù üåç

Components + Zone.js + Change Detection

  • Angular uses Zone.js to know when async events happen (click, HTTP, etc.).
  • Then it runs Change Detection over the entire component tree.
  • Default mode ‚Üí check everything
  • OnPush mode ‚Üí check only components with changed @Input() or manually marked
  • To force updates, we used detectChanges() or markForCheck()
‚úÖ Works fine
‚ùå But checks too much ‚Üí can slow down large apps
‚ùå Relies heavily on Zone.js (a global monkey patch)
‚ùå Hard to optimize ‚Äî too much auto-magic

‚ö° 2Ô∏è‚É£ With Signals: The ‚ÄúNew World‚Äù üöÄ

Instead of Angular checking everything,
Signals tell Angular directly when and what changed.

  • No Zone.js.
  • No global change detection
  • No need for detectChanges() in most cases
  • No need for Default/OnPush configuration
  • Everything happens automatically & precisely üéØ

Signals are the modern replacement for the whole reactive + CD + Zone system

But... Angular still supports the old system (for now)
