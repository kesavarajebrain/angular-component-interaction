Difference between this multiple forms?

| Feature        | Template | Reactive    | Signal    |
| -------------- | -------- | ----------- | --------- |
| Logic location | HTML     | TS          | TS        |
| State type     | ngModel  | FormControl | signal    |
| Validation     | HTML     | Validators  | computed  |
| Reactivity     | implicit | RxJS        | signals   |
| Boilerplate    | low      | high        | medium    |
| Debugging      | hard     | medium      | easy      |
| Performance    | ok       | good        | excellent |

        
1️⃣ Template-driven : HTML driven
2️⃣ Reactive Forms : FormGroup driven
3️⃣ Signal Forms : State driven

You said:
“validation per field is confusing”

That’s because:
Reactive Forms → validation centralized
Signal Forms   → validation distributed
| Feature      | Reactive Forms | SignalGroup API   |
| ------------ | -------------- | ----------------- |
| FormGroup    | FormGroup      | signalGroup       |
| FormControl  | FormControl    | signalControl     |
| Validators   | Validators     | signal validators |
| valueChanges | observable     | signals           |

            
👉 This looks like:
Reactive Forms + Signals combined
