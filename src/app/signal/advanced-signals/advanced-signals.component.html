1️⃣ resource() – Signals for Async Data (HTTP)

Problem Before Signals
Fetching API data in Angular traditionally:

this.http.get('/api/tasks').subscribe(tasks => {
  this.tasks = tasks;
});
                
Problems:
manual subscribe
manual loading state
manual error handling
unsubscribe issues
With resource()

Angular created resource() to manage async data with signals.

    It automatically handles:
  • loading
  • error
  • data
  • re-fetch
tasksResource = resource({
    loader: async () => {
        const response = await fetch(this.apiService.mockTaskUrl);
        console.log('response',response)
        return response.json();
    }
});

2️⃣ linkedSignal() – Dependent Signals

This is used when one signal depends on another signal’s state.

// get todos by id

    setSelectedTask(id: number) {
        this.selectedTaskId.set(id);
    }

// linked signal

    selectedTaskId = signal < number | null>(null);

    selectedTask = linkedSignal({
        source: () => ({
            todos: this.todos(),
            id: this.selectedTaskId()
        }),
        computation: ({ todos, id }) =>
            todos.find(t => t.id === id) ?? null
    });
