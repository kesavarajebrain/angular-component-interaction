Signal Practical Example

What exact signals? is the same like redux or ngrx?

Signals can replace NgRx in small/medium apps, but not always in very large enterprise apps.

Feature Signals useState NgRx
scope component/service component global
complexity simple simple complex
best for UI + shared state UI state large app state
  • Step 1 — Create Signal Store (Service) task.store.ts - similar to NgRx Store State
  • Step 2 — Create Computed Selectors - Just like NgRx selectors.
  • Step 3 — Add Store Actions - Equivalent to NgRx actions + reducers.
  • Step 4 — Use Store in Component
  • Step 5 — Template UI

Now our signal store looks like: 

State
signals()

Selectors
computed()

Actions
methods()
Exactly like NgRx but 10x simpler.
What Happens Internally? 


User adds task:

addTask()
   ↓
tasks.update()
   ↓
visibleTasks recompute
pendingCount recompute
completedCount recompute
   ↓
Error displaying
   ↓
UI updates automatically
            
No:
subscribe()
dispatch()
selectors boilerplate

Why This Pattern is Powerful?
Benefits:
✔ No RxJS complexity
✔ No NgRx boilerplate
✔ Automatic UI updates
✔ Simple debugging
✔ Type safe

Large companies now use: Signal Store Pattern
instead of heavy NgRx in many cases.
