Routing

Routing is not just navigation anymore:

Route = Navigation + Security + Data + Control + UX
1. Navigation from HTML (Template)

🔹 routerLink (MOST USED)
< a routerLink="/home">Go to Home </a> 👉 Click → navigates to /home

🔹 Dynamic route:
<a [routerLink]="['/user', userId]">User </a> 👉 If userId = 5 → /user/5

🔹 Query params:
<a [routerLink]="['/search']" [queryParams]="{ q: 'angular' }"> Search </a>👉 URL: /search?q=angular

🔹 Active link highlight:
<a routerLink="/home" routerLinkActive="active">Home</a> 👉 Adds class when active

2. Navigation from TypeScript (Programmatic)

✅ Import Router
import { Router } from '@angular/router';
constructor(private router: Router) {}

🔹 Method 1: navigate()
this.router.navigate(['/home']);

🔹 With params:
this.router.navigate(['/user', 10]); 👉 /user/10

🔹 With query params:
this.router.navigate(['/search'], { queryParams: { q: 'angular' } });

🔹 Relative navigation:
this.router.navigate(['details'], { relativeTo: this.route }); 

Normally when you do:
this.router.navigate(['/details']);

👉 Angular treats it as absolute path
👉 It always starts from root (/)

✅ Relative Navigation means:
“Navigate based on the current route”

Current URL: /forms

Now you write:
this.router.navigate(['reactive'], { relativeTo: this.route });

👉 Result: /forms/reactive
relativeTo → current route-ஐ base ஆக்கி navigate பண்ணு

🔹 Replace URL (no back history):
this.router.navigate(['/login'], { replaceUrl: true }); 

Normally:
this.router.navigate(['/home']);

👉 Browser history: /login → /home
👉 If user clicks BACK: /home → /login ❌

✅ With replaceUrl
this.router.navigate(['/home'], { replaceUrl: true });

👉 It replaces current URL in history
/login → replaced by /home

👉 Now BACK button:
/home → (no login page) ✅

🔐 Login flow:
loginSuccess() {
  this.router.navigate(['/dashboard'], { replaceUrl: true });
}
👉 Prevent user from going back to login page

🔹 navigateByUrl()
this.router.navigateByUrl('/home'); 👉 Simple string-based navigation
